Examples

Real-world implementation patterns for Navio.

Simple SPA with Hash Routing

The simplest way to use Navio. Perfect for static sites hosted on GitHub Pages or basic file servers where server-side rewrites aren't available. 

<!-- index.html -->
<script src="https://cdn.jsdelivr.net/gh/fahmirizalbudi/navio@master/bundle.js"></script>

<nav>
  <!-- Use standard paths, Navio handles the hash -->
  <a href="/" data-navio>Home</a>
  <a href="/about" data-navio>About</a>
</nav>

<div id="app"></div>

<script>
  const router = new Navio({ hashMode: true });
  const app = document.getElementById('app');

  router
    .on('/', () => app.innerHTML = '<h1>Home</h1>')
    .on('/about', () => app.innerHTML = '<h1>About</h1>')
    .start();
</script>

Dynamic Dashboard with Parameters

Fetching data based on URL parameters. This example simulates loading user profiles. 

const router = new Navio();

router.on('/users/:id', async (context) => {
  const userId = context.params.id;
  document.body.innerHTML = 'Loading user...';
  
  try {
    // Simulate API call
    const user = await fetchUser(userId); 
    
    document.body.innerHTML = `
      <h1>${user.name}</h1>
      <p>Email: ${user.email}</p>
      <a href="/users" data-navio>Back to List</a>
    `;
  } catch (err) {
    document.body.innerHTML = 'User not found';
  }
});

Loading External Views

Keep your code clean by separating HTML templates into files and loading them on demand. 

router.on('/contact', async () => {
  // Fetch 'contact.html' and inject body content
  const html = await Navio.fetchView('views/contact.html');
  document.getElementById('main').innerHTML = html;
  
  // Initialize specific logic for this view
  initContactForm(); 
});
